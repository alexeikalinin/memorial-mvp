# /idea — Добавить идею в список

Скилл для быстрого добавления идей в `docs/IDEAS.md` — из текущего разговора, мозгового штурма или анализа кода.

## Что делает скилл

1. Читает `docs/IDEAS.md`
2. Разбирает аргумент пользователя (название / описание идеи)
3. Если идея похожа на существующую — уточняет или дополняет её
4. Если новая — добавляет блок в нужный раздел
5. Добавляет строку в таблицу «Идеи из обсуждений» с текущей датой
6. Подтверждает добавление

## Формат вызова

```
/idea "Memory Graph View — граф воспоминаний как в Obsidian"
/idea добавить экспорт мемориала в PDF
/idea Stripe биллинг — уже есть в списке, обнови статус на planned
```

## Инструкция для Claude

Получив вызов `/idea`:

1. **Прочитай** `docs/IDEAS.md`
2. **Проверь** нет ли уже похожей идеи (по ключевым словам из аргумента)
   - Если есть — предложи обновить статус или дополнить описание, не дублируй
   - Если нет — создай новый блок
3. **Определи раздел** куда добавить:
   - Новая фича для пользователей → «Продуктовые идеи»
   - Технический долг / баг → «Технический долг»
   - Инструменты команды / процесс → «Продуктовые идеи» с пометкой «для команды»
4. **Создай блок** по шаблону:

```markdown
### [Название идеи]
**Статус:** `idea`
**Откуда:** обсуждение [YYYY-MM-DD]
**Описание:** [Краткое описание, 1-3 предложения]
**Технический базис:** [что уже есть в коде, если применимо]
**Ценность:** [зачем это нужно]
```

5. **Добавь строку** в таблицу «Идеи из обсуждений»:
```
| YYYY-MM-DD | Название | `idea` |
```

6. **Ответь пользователю** одной строкой: что добавлено / что обновлено.

## Когда использовать без явного вызова

Если в разговоре обсуждается новая фича, продуктовая идея или технический паттерн — предложи добавить в IDEAS.md. Например: «Хочешь записать эту идею в `docs/IDEAS.md`?»
