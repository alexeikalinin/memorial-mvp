## ФАЗА 1 — Первый запуск тестов

```bash
cd {BACKEND_DIR}
source .venv/bin/activate
python -m pytest tests/ -v --tb=long --no-header 2>&1
```

Сохрани в памяти:
- `initial_failures` — список упавших тестов с полными traceback
- `initial_score` — "PASSED/TOTAL"

---

## ФАЗА 2 — Анализ и фикс (повторяй до MAX_ITERATIONS или пока есть падения)

### Для каждого упавшего теста:

1. **Прочитай код теста** — понять что именно он проверяет
2. **Прочитай traceback** — найти строку где падает
3. **Прочитай production-код** который вызывается тестом
4. **Сформулируй гипотезу** — почему тест падает:
   - Баг в production-коде → исправить production-код
   - Тест неправильно написан (e.g. устаревший endpoint) → исправить тест
   - Фикстура не подходит (e.g. отсутствует auth) → исправить фикстуру
5. **Примени фикс** используя Edit или Write
6. **Запиши что изменил** в список `fixes_applied`

### Правила фикса:
- **Приоритет 1:** исправлять баги в production-коде (backend/)
- **Приоритет 2:** исправлять тесты если они неверно написаны
- **Не трогать:** логику бизнес-правил без понимания контекста
- **Не трогать:** migrations или схему БД без явной необходимости
- **Осторожно с:** auth.py, models.py — только точечные правки

## ФАЗА 3 — Ретест после фикса

```bash
cd {BACKEND_DIR}
source .venv/bin/activate
python -m pytest tests/ -v --tb=short --no-header 2>&1
```

Сравни с предыдущим запуском:
- Если тестов упало меньше → прогресс, продолжить
- Если упало столько же или больше → откатить последний фикс, попробовать другой подход
- Если все прошли → перейти к ФАЗЕ 4

**Повторяй ФАЗЫ 2-3 до MAX_ITERATIONS раз.**
