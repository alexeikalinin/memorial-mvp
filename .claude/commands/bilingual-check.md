# /bilingual-check — Проверь двуязычность реализованной фичи

Этот скилл запускается после реализации любой фичи или изменения UI.
Он проверяет, что изменения работают одинаково для **обоих языков** (EN и RU).

## Контекст проекта

- Мемориалы бывают двух языков: **EN** (English) и **RU** (Русский).
- Сейчас RU-интерфейс **скрыт** в нескольких местах специально для демо инвесторам (англоязычная аудитория). Скрытие реализовано через условия типа `lang === 'ru'` или `language === 'ru'`.
- Это **временное** скрытие — функционал должен работать для обоих языков. Пользователь будет сам выбирать язык мемориала.
- Нельзя делать фичи только для EN и забывать про RU.

## Что проверять

### 1. Найди места скрытия RU-контента
```bash
grep -rn "lang === 'ru'\|language === 'ru'\|lang !== 'en'\|language !== 'en'\|SHOW_RU\|ru.*hidden\|hidden.*ru" \
  /Users/alexei.kalinin/Documents/VibeCoding/memorial-mvp/frontend/src/ \
  --include="*.jsx" --include="*.js" --include="*.css"
```

### 2. Проверь переводы

Для каждого нового текстового элемента убедись, что он есть в обоих файлах локализации:
```bash
ls /Users/alexei.kalinin/Documents/VibeCoding/memorial-mvp/frontend/src/locales/
# Обычно: en.json и ru.json
```

Найди ключи, которые есть в EN, но нет в RU (и наоборот):
```bash
# Ключи из en.json
cat frontend/src/locales/en.json | python3 -c "import json,sys; d=json.load(sys.stdin); [print(k) for k in d.keys()]"
```

### 3. Проверь API-эндпоинты

Убедись что backend обрабатывает оба языка одинаково:
```bash
grep -n "language\|lang\|'ru'\|'en'" \
  /Users/alexei.kalinin/Documents/VibeCoding/memorial-mvp/backend/app/api/*.py
```

### 4. Проверь компоненты

Для каждого изменённого компонента проверь:
- Нет ли `if (lang === 'en')` блоков, которые исключают RU
- Новые props/state работают независимо от языка
- CSS-классы не зависят от языка

## Чеклист после проверки

Выведи результат в формате:

```
═══════════════════════════════════════════
  BILINGUAL CHECK — {компонент/фича}
═══════════════════════════════════════════

✅ Работает для EN: да
✅ Работает для RU: {да | нет — см. ниже}

Временно скрытые RU-места (OK, это нормально):
  - {файл:строка} — {причина скрытия}

⚠️  Проблемы (нужно исправить):
  - {описание проблемы}

Отсутствующие переводы:
  - {ключ} — есть в EN, нет в RU
  - {ключ} — есть в RU, нет в EN

═══════════════════════════════════════════
ВЫВОД: {✅ ВСЁ ОК | ⚠️ ТРЕБУЕТ ДОРАБОТКИ}
```

## Важно

- **Не убирай** временные скрытия (`lang === 'ru'`) — это сознательное решение для демо.
- **Добавляй переводы** для всех новых строк в оба файла локализации.
- **Не делай логику зависимой от языка** — только UI/отображение может быть локализовано, бизнес-логика должна быть единой.
- Если фича не работает для RU — зафиксируй это в `SESSION_LOG.md` как незавершённую задачу.
