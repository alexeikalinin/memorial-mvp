Выполни полный чеклист готовности memorial-mvp к деплою в production.

Прочитай файлы:
- backend/.env или backend/.env.example
- backend/app/config.py — все настройки
- backend/app/main.py — CORS и регистрация роутеров
- backend/app/api/memorials.py — найди все места с owner_id=1 (заглушка вместо аутентификации)

Проверь каждый пункт:

**Безопасность**
- [ ] SECRET_KEY: не дефолтный ("dev-secret-key..."), длина > 32 символов
- [ ] DEBUG=false
- [ ] CORS_ORIGINS: только нужные домены, не "*"
- [ ] owner_id=1 хардкод — предупреди: API полностью открыт, любой может создавать/читать мемориалы

**Хранилище**
- [ ] USE_S3=true — файлы в S3, не на локальном диске (локальный диск не персистентен в большинстве хостингов)
- [ ] S3_BUCKET_NAME, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY заполнены
- [ ] Или есть другое решение для персистентного хранилища

**База данных**
- [ ] DATABASE_URL не SQLite (для production нужен PostgreSQL)
- [ ] Есть механизм бэкапа БД

**AI-сервисы**
- [ ] OPENAI_API_KEY заполнен
- [ ] DID_API_KEY или HEYGEN_API_KEY заполнен (если нужна анимация)
- [ ] ELEVENLABS_API_KEY заполнен (если нужен TTS)
- [ ] PUBLIC_API_URL указывает на реальный публичный URL (не localhost, не ngrok)

**Очереди задач**
- [ ] Redis доступен в production (не localhost)
- [ ] Celery worker настроен как отдельный процесс/контейнер

**Тесты**
- Запусти: cd backend && python -m pytest -v
- Покажи результаты
