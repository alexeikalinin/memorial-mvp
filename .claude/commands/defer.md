# /defer — Отложить предложенное, но не реализованное

Скилл для сохранения идей, фич и решений, которые **обсуждались в разговоре, но не были реализованы** — чтобы не потерять и вернуться позже.

## Когда использовать

- Предложил что-то хорошее, но решили «не сейчас»
- Нашли проблему, которую пока не фиксим
- Появилась идея улучшения в процессе работы над другой задачей
- Обсудили архитектурное решение, которое отложили

## Что делает скилл

1. Читает `docs/IDEAS.md`
2. Определяет нужный раздел по контексту:
   - 🔴 Launch Blocker → раздел "Launch Blockers"
   - UX/продукт → "Core UX" или "Продуктовые идеи"
   - Технический долг → "Технический долг"
   - Остальное → "Идеи из обсуждений (лог)"
3. Добавляет блок с полями:
   - **Статус:** `deferred`
   - **Откуда:** текущая дата + краткий контекст из разговора
   - **Описание:** что предложено и почему отложено
   - **Вернуться когда:** условие или срок
4. Добавляет строку в таблицу лога внизу
5. Подтверждает сохранение

## Формат вызова

```
/defer "добавить biography agent в UI мемориала — API есть, отложили до онбординга"
/defer rate limiting — slowapi, откладываем до первых 100 пользователей
/defer "14-day trial без карты — обсудили, реализация после Stripe e2e"
```

## Формат записи в IDEAS.md

```markdown
### [Название]
**Статус:** `deferred`
**Откуда:** обсуждение YYYY-MM-DD — [контекст]
**Описание:** [что предложено]
**Вернуться когда:** [условие / milestone]
```

## Правила

- Не удаляй и не переписывай существующие записи — только добавляй
- Если похожее уже есть → обнови статус на `deferred` и добавь "Вернуться когда"
- Всегда добавляй строку в таблицу "Идеи из обсуждений (лог)" с датой
- Дату брать из системного контекста (today's date)
