# E2E-чеклист: семейный RAG (`include_family_memories`)

**Условия прогона:** backend `http://127.0.0.1:8000`, в `.env` заданы `OPENAI_API_KEY`, Qdrant (cloud), SQLite с сидом семьи Морозовых/Ковалёвых.  
**Запрос:** `POST /api/v1/ai/avatar/chat` с телом:

```json
{
  "memorial_id": <ID аватара B>,
  "question": "<текст>",
  "include_audio": false,
  "include_family_memories": true,
  "use_persona": true
}
```

**Логика пары:** воспоминание лежит у мемориала **A**, в тексте упомянут **B**; в `family_relationships` есть строка `memorial_id = B`, `related_memorial_id = A` — тогда при чате с **B** в поиск попадают воспоминания **A**.

Дата прогона: **2026-03-25** (ответы зафиксированы с реального API).
