# Паритет локально ↔ веб (демо EN)

В репозитории **нет** файла `memorial.db` (он в `.gitignore`). «Взять мемориалы из git» как готовую БД **нельзя** — в git только **код сидов** (`seed_english_all.py` и цепочка). Идентичность данных достигается **одинаковым порядком сидов** и **одинаковой конфигурацией** API и медиа.

## 1. Одна и та же граф семьи

1. Сбросьте локальную SQLite (или очистите таблицы на Postgres, если сидите туда осознанно).
2. Из каталога `backend/` выполните **ровно**:
   ```bash
   source .venv/bin/activate
   SEED_USE_SQLITE=1 python seed_english_all.py
   python verify_en_demo_graph.py
   ```
3. Скрипт `verify_en_demo_graph.py` проверяет: 43 EN-имени из `en_memorials_manifest.py`, родителей у ключевых персон (Kelly / Chang / Rossi) и браки (Michael↔Catherine, James↔Helen, Emily↔Daniel и т.д.).

Если после правок сидов проверка падает — обновите список проверок в `verify_en_demo_graph.py`.

**Важно:** в `seed_english_cluster2.py` рёбра `PARENT`/`CHILD` должны соответствовать API (`app/api/family.py`): **PARENT** — memorial = ребёнок, related = родитель; **CHILD** — memorial = родитель, related = ребёнок. Иначе на проде и локально дерево будет «ломаться» одинаково неправильно.

## 2. Одни и те же картинки

- Дерево и страница мемориала используют **`cover_photo_id`** из API.
- Локально без S3 файлы лежат в `backend/uploads/`; на проде при `USE_S3=true` — в объектном хранилище.
- Чтобы портреты EN-демо совпадали с манифестом имён: `clear_en_demo_covers.py` → `seed_english_portraits.py` (нужна сеть к randomuser).

## 3. Фронт смотрит на тот же бэкенд

| Окружение | Что настроить |
|-----------|----------------|
| Локально | Vite прокси: не задавать `VITE_API_URL` или `frontend/.env.local` с URL локального API. |
| Прод | `VITE_API_URL=https://<ваш-бэкенд>/api/v1` на этапе **сборки** Vercel. |

Если в браузере открыт прод-фронт, а вы правите только локальный `memorial.db`, вы **всегда** будете видеть «другое дерево» — фронт ходит на **удалённый** API.

## 4. Прод-БД

Чтобы прод совпадал с локальным демо после деплоя:

- Либо выполнить на сервере тот же `seed_english_all.py` (к моменту, когда `DATABASE_URL` указывает на прод Postgres),
- Либо один раз залить дамп, собранный **после** успешного `verify_en_demo_graph.py` на эталонной машине.

Подробнее про переменные: [ENVIRONMENT.md](../ENVIRONMENT.md).
