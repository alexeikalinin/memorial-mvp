# Family tree — раскладка и коннекторы (режим «поколения»)

Краткая «точка правды» для разработки. Подробные итерации и таблица замечаний — в [SESSION_LOG.md](../SESSION_LOG.md) (поиск по «Family tree (поколения)» / дата).

## Где в коде

| Область | Файл |
|--------|------|
| Размеры карточки, колонки, зазоры, полосы поколений | `frontend/src/utils/familyTreeGenerationLayout.js` |
| Линии, брачная горизонталь, вилка parent→child, кольца | `frontend/src/utils/familyTreeOrthogonalConnectors.js` |
| Рендер карточек и SVG | `frontend/src/components/FamilyTree.jsx` |
| Стили gen-карточек (рамка кластера, тень) | `frontend/src/components/FamilyTree.css` (`.ft-node--gen`) |

Режим сетки (не поколения) использует **другие** константы в `FamilyTree.jsx` (`CELL_W`/`CELL_H`, `NODE_W`/`NODE_H` для `NodeCard`) — это отдельная раскладка; режим поколений берёт размеры из `buildGenerationLayout().nodeSize`.

## Константы (поколения)

Значения задаются в `familyTreeGenerationLayout.js` (не дублировать в JSX):

- **`NODE_W` / `NODE_H`** — логический прямоугольник узла; совпадает с `genLayout.nodeSize` и с коннекторами (`nodeSize` в `buildOrthogonalConnectors` / `getSpouseMarriageMarkers`).
- **`GAP_X`** — горизонтальный зазор между двумя карточками в одной строке.
- **`SUB_ROW_GAP`** — вертикаль между перенесёнными строками **внутри** одного поколения (несколько «линий» в колонке).
- **`INTER_GEN_GAP`** — вертикаль **между этажами поколений**. Нужен, потому что у кластерных карточек большой `box-shadow`: при нулевом зазоре тень верхнего ряда визуально лежит на нижнем.
- **`LINEAGE_LANE_GAP`** — расстояние между колонками фамилий (A / center / B / …).
- **`ROW_H`** — опорная высота для эвристик в коннекторах (вилка и т.д.), когда явный геометрический зазор мал.

Брачная линия и кольца: горизональ только в зазоре между карточками (`marriageBarInGap`); при нулевом зазоре по X — отрезок ненулевой ширины вокруг середины, чтобы кольца не схлопывались в точку. При разном вертикальном положении супругов горизонталь на средней высоте между «уровнями» брака на карточках, с вертикальными стыками к ней.

**Длинная линия / «кольца в пустоте»:** супруги из разных колонок (Kelly слева, Anderson справа и т.д.) соединяются **одной** горизонталью на всю ширину между их карточками; иконка колец стоит в **середине** сегмента. В кадре может быть виден только один человек — второй супруг в другой колонке или за краем экрана; при наведении на кольца показывается tooltip с **именами обоих** (поле `pairTitle` в `getSpouseMarriageMarkers`). У `.ft-marriage-rings` включены `pointer-events`, чтобы подсказка работала.

**Двухцветная рамка карточки (например Helen … Anderson Kelly):** это **две половины цветов кластеров фамилий** из легенды (Kelly / Anderson и т.д.), а не отдельная «линия родства к зелёной семье». Связи смотрите по рёбрам графа (parent/child/spouse). Подробнее — строки `split_border_title` и `gen_legend_help_borders` в `frontend/src/locales/*.js`.

**Один «этаж» ≠ одна семья:** узлы сортируются по **номеру поколения** из API и колонкам фамилий, а не по «поддереву». Пара Daniel+Emily может оказаться **визуально под** парой Serena+Luca, потому что у них **один уровень поколения** в графе, а не потому что они дети Rossi/Chang сверху.

## Портреты EN-демо

Дерево и карточка мемориала показывают **`cover_photo_id`** из API. Неверный пол на фото после сидов: переназначение обложек по имени — `backend/seed_english_portraits.py` и `portrait_params_en.py`; для сброса старых привязок см. `clear_en_demo_covers.py` в репозитории (и записи в `SESSION_LOG` про EN seed).

Чтобы локальный UI и прод показывали **одни и те же** данные и картинки, см. [DEMO_PARITY.md](DEMO_PARITY.md).
