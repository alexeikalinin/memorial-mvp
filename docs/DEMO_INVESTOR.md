# Режим демонстрации инвестору

Кратко: как включить «полный доступ на всех демо-мемориалах» для любого залогиненного пользователя и где описан обычный продуктовый функционал.

## Переменные бэкенда (Railway)

| Переменная | Назначение |
|------------|------------|
| `INVESTOR_DEMO_MODE=true` | Любой пользователь с валидным JWT считается **owner** на **всех** мемориалах: список всех страниц, редактирование, медиа, воспоминания, инвайты, чат с аватаром, семейное дерево, панель **Access**. |
| `GLOBAL_ADMIN_EMAILS=...` | Только перечисленные email получают тот же эффект; с демо режимом можно использовать вместе или вместо него. |

**Важно:** перед реальным запуском выключите `INVESTOR_DEMO_MODE` (и при необходимости сузьте `GLOBAL_ADMIN_EMAILS`).

## Публичные страницы без входа

Чтобы карточки открывались по публичным ссылкам `/m/:id`, у мемориала должно быть `is_public=true`. Массово на прод-БД:

```bash
cd backend && source .venv/bin/activate && python set_all_memorials_public.py
```

(`DATABASE_URL` должен указывать на ту же БД, что и Railway.)

## Выход из аккаунта

В шапке SPA отображаются email и кнопка **Log out / Выйти** (очищает `localStorage` и ведёт на `/login`).

### После одобрения инвестора

1. Выключить `INVESTOR_DEMO_MODE`.
2. Удалить демо-данные (мемориалы) — вручную или скриптом под вашу БД.
3. Новые пользователи создают свои мемориалы; доступ — через **Access** (роли owner/editor/viewer), **инвайт-ссылки** для вклада без регистрации, **запрос доступа** к приватным страницам (см. тесты и `access` API).
