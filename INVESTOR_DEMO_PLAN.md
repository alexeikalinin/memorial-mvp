# План демонстрации для инвесторов — Memorial MVP

## Главный тезис

**Мы сохраняем личности, а не просто фотографии.**

Memorial MVP — это сервис сохранения памяти об ушедших людях через цифровые мемориалы, коллекцию воспоминаний и ИИ-аватара, который отвечает на вопросы на основе этих воспоминаний. Цель — дать семьям и близким возможность не только хранить фото и даты, но и **восстанавливать образ человека**: как он говорил, что любил, что переживал, как относился к близким.

---

## 1. Что это за приложение

- **Цифровой мемориал** — страница человека с фото, видео, текстовыми воспоминаниями и **чатом с ИИ-аватаром**, который говорит от его имени, опираясь на загруженные воспоминания (RAG).
- **Для кого:** семьи, друзья, сообщества — все, кто хочет сохранить и передать память о человеке и иметь возможность «поговорить» с ним через аватар.
- **Зачем:** чтобы память не ограничивалась альбомом и датами, а оставалась **живой** — в рассказах, в голосе, в ответах на вопросы потомков и близких.

---

## 2. Ключевой функционал (что показывать на демо)

### 2.1 Создание мемориала и наполнение контентом
- Создание страницы мемориала (имя, описание, даты, обложка).
- Загрузка медиа: фото и видео с автоматическими превью и миниатюрами.
- Добавление **воспоминаний** — текстовые истории с опциональной датой события; они индексируются в векторной БД (Qdrant/Pinecone) для умного поиска в чате.

### 2.2 Чат с ИИ-аватаром (RAG)
- Пользователь задаёт вопрос — система находит релевантные фрагменты воспоминаний и генерирует ответ от лица человека.
- Ответы **только на основе загруженных воспоминаний** — без выдумок; если информации нет, аватар честно говорит об этом.
- Опционально: озвучка ответа голосом (ElevenLabs), в том числе **кастомный голос** по загруженной записи — аватар говорит «его» голосом.

### 2.3 Оживление фотографий (D-ID / HeyGen)
- Любое фото мемориала можно «оживить»: ИИ генерирует короткое видео, где человек на фото говорит (по заданному тексту или приветствию).
- Подчёркивает идею «личность, а не просто фото» — образ оживает.

### 2.4 Семейное дерево
- Связи между мемориалами: супруг/супруга, дети, другие родственники.
- Визуализация дерева с понятными бейджами (ребёнок, супруг/а, страница) и отдельным визуальным выделением умерших.
- Навигация между мемориалами семьи в одном контексте.

### 2.5 Хронологическая лента (Life Timeline)
- Воспоминания с датой события собираются в ленту по времени — жизнь человека в ключевых моментах.

## 3. Крутые фишки (акценты для инвесторов)

### 3.1 Синхронизация памяти между участниками мемориала и семьёй
- **Суть:** если в воспоминании одного человека (например, дедушки) упоминается другой из мемориала или семейного дерева (например, бабушка), система может **синхронизировать** это воспоминание в мемориал бабушки.
- **Memory Sync Agent** находит упоминания родственников в тексте, создаёт «отражённые» воспоминания (с пометкой источника) в мемориалах этих родственников и индексирует их. В результате **оба аватара** — и дедушки, и бабушки — могут отвечать на вопросы про общие события, хотя изначально воспоминание было добавлено только в один мемориал.
- В чате есть опция «Воспоминания родственников» — при включении RAG ищет по всем связанным мемориалам, и кнопка «Синхр. с семьёй» запускает синхронизацию.
- **Почему это сильно:** память становится **общей семейной**, а не разрозненной по страницам; следующий уровень после «просто чата с одним аватаром».

### 3.2 Публичная страница и шаринг по ссылке (включая план по упрощённому добавлению воспоминаний)
- У каждого мемориала есть **публичная страница** по ссылке вида `/m/{id}` — без входа и регистрации.
- На ней доступны: просмотр воспоминаний, фотографий и **чат с аватаром**. Можно скачать **QR-код** этой страницы (например, для размещения на памятнике или в памятных местах).
- **Реализовано:** поделиться ссылкой, открыть страницу, пообщаться с чатом, посмотреть контент — всё без лишних регистраций и доступов.
- **В планах / в процессе:** упрощённый сценарий для тех, кто «не разбирается», но хочет внести вклад — по той же ссылке возможность **напечатать или надиктовать воспоминание** и отправить его в мемориал (минимальный интерфейс: поле ввода/голос + кнопка «Добавить»), без регистрации и сложного доступа. Это превращает ссылку в инструмент **коллективного сбора памяти**.

### 3.3 Умная персона аватара (Smart Avatar Persona)
- Системный промпт аватара **строится автоматически** из всех воспоминаний мемориала — характер, привычки, стиль общения, факты жизни.
- При добавлении новых воспоминаний кэш персоны обновляется — аватар «учится» на новых историях.

### 3.4 Гибкая инфраструктура
- Векторная БД: **Qdrant** (в т.ч. локально) или **Pinecone**.
- Анимация фото: **D-ID** или **HeyGen** (переключается через конфиг).
- При недоступности Redis/Celery часть задач (например, создание эмбеддингов, в ряде сценариев анимация) выполняется синхронно — сервис остаётся рабочим.

## 4. Чем продукт помогает людям

- **Семьям:** единое место для памяти о близком, общие воспоминания и возможность «поговорить» с ним через аватар.
- **Потомкам:** доступ к историям и характеру предка не только через сухие даты и фото.
- **Друзьям и сообществам:** простой способ поделиться ссылкой и собирать воспоминания от многих людей (особенно с доработкой «добавить воспоминание по ссылке»).
- **Эмоциональная ценность:** снижение ощущения потери за счёт сохранения голоса, стиля общения и личных историй в интерактивной форме.

## 5. Структура рассказа на демо (краткий сценарий)

1. **Вступление (1–2 мин)**  
   Тезис: «Мы сохраняем личности, а не просто фотографии.» Кратко: что такое Memorial MVP и для кого.

2. **Живое демо (5–7 мин)**  
   - Показать мемориал: медиа, воспоминания, семейное дерево.  
   - Добавить воспоминание, где упоминается родственник из дерева.  
   - Запустить «Синхр. с семьёй» (или показать dry_run), пояснить синхронизацию памяти.  
   - Чат с аватаром: вопрос по своему воспоминанию; затем включить «Воспоминания родственников» и задать вопрос по общему событию — ответ от «другого» аватара.  
   - Показать публичную страницу по ссылке, QR-код; при желании — озвучку ответа и/или оживление фото.

3. **Фишки и дорожная карта (2–3 мин)**  
   - Синхронизация памяти между участниками и семейным деревом.  
   - Публичная ссылка: уже можно делиться и общаться с чатом; в планах — упрощённое добавление воспоминаний (напечатать/надиктовать) без регистрации.  
   - Умная персона, кастомный голос, лента жизни.

4. **Заключение (1 мин)**  
   Повтор тезиса и призыв: продукт даёт семьям и сообществам инструмент сохранять и оживлять личность — не только альбом и даты.

## 6. Технический контекст (на вопросы)

- **Стек:** Backend — Python 3.11, FastAPI, SQLAlchemy, SQLite/PostgreSQL; Frontend — React 18, Vite; ИИ — OpenAI (LLM + embeddings), D-ID/HeyGen, ElevenLabs; векторная БД — Qdrant или Pinecone; фоновые задачи — Celery + Redis с синхронным fallback.
- **Аутентификация и доступ:** регистрация и вход по JWT (Bearer); владелец мемориала задаётся через модель прав доступа (`MemorialAccess`), а не захардкоженный ID. Публичная страница `/m/:id` по-прежнему доступна без входа для просмотра и чата (настраиваемо политикой продукта). API для выдачи и отзыва прав, а также заявок на доступ (`AccessRequest`) реализован на backend; UI управления доступом в кабинете владельца — в разработке.
- **Масштабирование:** загрузка медиа на S3, векторная БД в облаке, разделение воркеров — заложено в архитектуре.
